# Technology Stack

## Architecture

単一バックエンドが SPA 配信・API・ActivityPub 連合・非同期タスク処理をすべて担うモノリス構成。外部依存はできる限り抑え、ランタイムは「アプリ + PostgreSQL」のみを目指す。

- **ローカル配送：意味論は対称、物理配送は最適化**。ローカル宛でも Activity の生成・可視性判定・状態遷移はリモートと完全に共通のコードパスを通す。分岐させるのは最終的な配送手段（in-process 関数呼び出し or HTTP 送信）のみ。
- ショートカット（in-process 配送）は JSON-LD シリアライズ／署名往復を通らないため、ローカルでは顕在化せずリモート連合でだけ壊れるバグを生みやすい。論理的に同一の Activity オブジェクトを生成・検証してから配送関数を分岐させる。

## Core Technologies

- **言語（バックエンド）**：Rust
- **Web フレームワーク**：axum
- **非同期ランタイム**：tokio
- **DB**：PostgreSQL
- **DB クエリ**：sqlx
- **非同期タスク処理**：DB キュー（外部のジョブブローカーに依存しない）
- **フロントエンド**：TypeScript + React
- **全文検索**：PostgreSQL 内で完結（外部検索エンジン不使用）

## Package Management

- Python を使う補助ツール等では `pip install` ではなく **uv** を使う（`uv venv` / `uv add` / `uv sync`）。

## Development Standards

### Testing — AI 自律 TDD が前提

Mastodon 互換 API は **Opus による自律 TDD**（契約固定 → 実装 → グリーン → レビューの垂直スライス自走）で実装する。コーディングのスループットはボトルネックではなく、**設計判断とレビューが律速**。テストの階層と責務を分ける：

- **単体**：ハンドラ・シリアライザ。
- **契約**：エンティティ（Status / Account / Notification / Poll …）の JSON 形をゴールデン／スナップショットで先に固定し、実装をそれに合わせる。
- **統合**：DB 込みのエンドポイント往復。
- **連合**：自前インスタンスを 2 つ起動して Activity 往復を検証。ローカル最適化パス（in-process）と HTTP 連合パスが**同じ結果**になることを必ずテストする。
- **E2E**：実クライアント（Ivory・Elk・Phanpy）のキャプチャ再生を最終受け入れ基準にする。

### 決定性の強制

時刻・ID・乱数・署名鍵は**注入可能（DI）**にする。clock / id generator / RNG / 署名鍵を差し替え可能な境界に置く。非決定的（flaky）なテストは自律ループを壊す。

### 仕様の出典・優先順位

一次情報は**実装の実挙動**（Mastodon 本体の実レスポンス）。**ドキュメント < 実レスポンス**で扱う。既存クライアントの実リクエストをキャプチャしてフィクスチャ化し受け入れ基準にする。

### 可観測性

失敗時にリクエスト／レスポンス／実行 SQL を十分に出力する。AI が原因特定・自己修正できる診断情報を欠かすとループが停滞・誤修正に陥る。

## Key Technical Decisions

- **配布形態**：事前ビルド済み（x86_64 / aarch64 Linux）。可能なら musl static 単一バイナリを第一候補とするが、真の要件は「インストールが簡単」であり単一バイナリには固執しない。メディア処理でネイティブ依存（libvips・ffmpeg 等）が要る場合は Docker イメージ等に切り替えてよい。
- **React 資産の埋め込み**：ビルド済みフロントをバイナリに埋め込み（`rust-embed` 等）、バックエンドが SPA を配信。Node も別 Web サーバーも不要。
- **DB マイグレーション**：埋め込み + 起動時自動実行（`sqlx migrate`）。up とデータ保持、失敗時の安全停止をテストする。
- **HTTPS**：デフォルトは内蔵 ACME（`rustls-acme` 等、Let's Encrypt 自動取得・更新・TLS 終端）。オプションでリバースプロキシ後段（`X-Forwarded-*` 尊重）。
- **設定の二層化**：起動設定（ドメイン・DB 接続・シークレット）は TOML / 環境変数。運用設定（インスタンス情報・トグル等）は DB 保存 + 管理画面編集。
- **検索の抽象化**：検索処理を抽象レイヤーの背後に置き、初期は標準 PostgreSQL の最小実装。`pg_bigm` 等の日本語拡張は任意オプションに留め、必須にしない。API 契約は Mastodon 検索 API の形に留める。
- **横断サブシステム**：OAuth 2.0 サーバー・Streaming API（WebSocket）・Web Push（VAPID）・ページネーション規約（`Link` + `max_id`/`since_id`/`min_id`）・エラー/レート制限互換（`X-RateLimit-*`）・非同期メディアアップロード（`202` → ポーリング）が API の前提。
